# Laboratorio: Optimización en Procesadores Segmentados (MIPS)

## Descripción
Este proyecto corresponde al laboratorio de **Estructura de Computadores**, donde se analiza el comportamiento de un programa en un procesador MIPS segmentado (pipeline) utilizando el simulador **MARS**.

Se identifican riesgos de datos (hazards), se optimiza el código y se comparan los resultados entre la versión base y la versión optimizada.

## Contenido del repositorio
- `programa_base.asm` → Código original del laboratorio.
- `programa_optimizado.asm` → Código optimizado para reducir stalls en el pipeline.
- `Plantilla_Informe.pdf` → Informe final del laboratorio.

## Herramientas utilizadas
- MARS 4.5
- MIPS X-Ray
- Instruction Counter
- Instruction Statistics
- Visual Studio Code
- GitHub

## Objetivos del laboratorio
- Identificar riesgos de datos (Load-Use).
- Analizar el comportamiento del pipeline.
- Optimizar el orden de instrucciones.
- Comparar rendimiento entre versiones del programa.
